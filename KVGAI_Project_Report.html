01 — Overview

Project Description

OmniMind Hub is a full-stack, AI-powered knowledge management and academic library platform. It enables users to upload various document formats (PDFs, Word documents, Markdown, and plain text) which are then automatically processed, summarized, categorized, and embedded into a high-dimensional vector space by the Google Gemini AI API in real time.

The platform features a Hybrid Semantic Search Engine that fuses traditional SQL keyword matching with deep cosine-similarity vector search powered by the pgvector PostgreSQL extension on Supabase. This allows users to find documents not just by exact keywords, but by meaning and conceptual relevance. For example, searching "symbolic reasoning" will surface documents about "formal logic" even if those exact words don't appear.

A personalized "For You" dashboard dynamically recommends topics by analyzing three data streams: the globally most-viewed categories, the user's own upload tags, and their historical search queries. The platform also enforces per-user upload limits, supports secure material deletion with ownership authorization, and provides original file downloads with inline previewing.

The entire application is deployed as a monorepo on Vercel, with the Next.js frontend and the Python FastAPI backend running as serverless functions behind a unified URL rewrite layer. The database is hosted on Supabase (managed PostgreSQL with pgvector). Note: Vercel serverless functions enforce a 5MB maximum file upload size per request.

03 — Architecture

System Architecture

OmniMind follows a monorepo architecture deployed on Vercel. The frontend is a Next.js 16 application using the App Router pattern, and the backend is a Python FastAPI application deployed as a Vercel Serverless Function. All API requests matching /api/* are rewritten to /api/main.py via the vercel.json configuration. The database is hosted on Supabase (managed PostgreSQL with the pgvector extension enabled).

Frontend Pages

  • Landing Page (/): Marketing page with animated hero, features showcase, testimonials carousel, live metrics counters, pricing section, and responsive navigation with mobile menu overlay.
  • Login / Signup (/login): Authentication page supporting email/password registration and login with JWT-based session management.
  • Dashboard (/dashboard): Authenticated home featuring a personalized "For You" carousel, most-visited materials, recent uploads, contribution counter (X/10 limit), and a global search bar.
  • Browse (/browse): Filterable material listing with sort options (Trending, Most Viewed, My Contributions, All), plus hybrid search integration.
  • Material View (/material/[id]): Deep-dive page showing AI-generated summary, full content, tags, view count, original file download, and related materials.
  • Upload (/upload): Drag-and-drop multi-file upload with per-file tag assignment, global "Apply Tag to All" injector, and real-time upload limit enforcement.

Backend Modules

  • main.py: Core FastAPI application with all route handlers, category seeding, database auto-migration, and the "For You" topic generation algorithm.
  • auth.py: JWT authentication module using bcrypt password hashing, HS256 token signing, and OAuth2 bearer token dependency injection.
  • ai_service.py: AI processing module handling text extraction (PDF via pypdf, DOCX via python-docx, TXT/MD), content analysis via Gemini 2.5 Flash, and 768-dimensional embedding generation via text-embedding-004 with REST API fallback.
  • models.py: SQLAlchemy ORM models for Users, Categories, Materials, MaterialViews, and SearchHistory.
  • schemas.py: Pydantic v2 request/response schemas with email validation and ORM-compatible serialization.
  • database.py: Supabase PostgreSQL engine configuration using NullPool strategy optimized for serverless execution environments.

MethodRouteDescription
POST /api/materials Upload files (PDF, DOCX, TXT, MD). Enforces 10-doc limit per user. Extracts text, generates AI summary/tags/category via Gemini, creates 768-dim embedding for semantic search, stores original file bytes. Accepts per-file tags as a 2D JSON array.

DELETE /api/materials/{id} Delete a material. Ownership authorization enforced: only the uploader can delete. Cascades deletion to associated MaterialViews.
GET /api/materials/{id}/download/original Download original uploaded file bytes. Supports inline preview mode via ?inline=true query parameter.

Browse & Discovery

MethodRouteDescription
GET /api/materials/browse/most-viewed Returns top 10 materials ranked by view count (descending).
GET /api/materials/browse/all Returns all materials with pagination, ordered by creation date.
GET /api/dashboard/stats Returns personalized dashboard data: most visited materials, recent uploads, user contribution count, and AI-generated "For You" trending topics derived from global category views, user upload tags, and search history.

Search

MethodRouteDescription
GET /api/search?q={query} Hybrid search engine. Logs the query for personalization, generates a query embedding, performs cosine-distance vector search via pgvector, simultaneously executes keyword search across title/summary/content, then merges results with text/title matches prioritized first. Falls back to pure text search if embedding generation fails.

06 — Functionality

Key Features In Depth

1. AI-Powered Document Processing Pipeline

When a user uploads a file, the backend orchestrates a multi-stage processing pipeline:

  • Text Extraction: Files are parsed based on their extension: pypdf for PDFs, python-docx for Word documents, and UTF-8 decoding for plaintext/Markdown files.
  • AI Analysis: The extracted text (capped at 10,000 characters) is sent to Gemini 2.5 Flash with a structured JSON prompt requesting a summary, up to 5 tags, and a category slug from a predefined taxonomy of 14 categories.
  • Embedding Generation: A rich "Advanced Semantic Text" block is constructed by concatenating the file title, AI-generated tags, summary, and content. This composite string is embedded into a 768-dimensional vector via the text-embedding-004 model, with a REST API fallback if the SDK fails.
  • Tag Merging: User-provided manual tags (sent as a 2D JSON array for per-file granularity) are merged with AI-generated tags, with user tags taking priority and duplicates removed.
  • Storage: The original file bytes are stored in the database alongside the processed metadata, enabling original file downloads.

2. Hybrid Semantic Search Engine

The search endpoint implements a dual-path retrieval strategy:

  • Semantic Path: The query string is embedded into a 768-dim vector, then a cosine-distance search is executed against all material embeddings using pgvector's built-in operators. This finds conceptually related documents even without keyword overlap.
  • Text Path: Simultaneously, a SQL ILIKE query searches across title, summary, and content columns for keyword matches. Results are ranked using a CASE expression: title matches rank first (priority 1), summary matches second (priority 2), and content matches third.
  • Fusion: Text matches (higher precision) are placed first in the result set, followed by semantic matches (higher recall), with deduplication by material ID.
  • Personalization: Every search query is logged to the search_history table, feeding into the "For You" recommendation engine.

3. Personalized "For You" Recommendation Engine

The dashboard's recommendation carousel is powered by a three-source algorithm:

  • Source 1: Global Trends: Aggregates total view counts grouped by category, surfacing the most popular topic platform-wide.
  • Source 2: User Upload Tags: Analyzes the authenticated user's uploaded materials, extracts tag frequency counts, and promotes the user's top 3 most-used tags as recommended topics.
  • Source 3: Search History: Mines the user's recent search queries, tokenizes them, filters stop-words, and surfaces the most frequently searched keywords.
  • Fallback: If insufficient data exists (new user), the system gracefully falls back to predefined default topics (AI & Machine Learning, Data Science, Software Engineering).

4. Dashboard Performance Optimization

  • Query Optimization: Dashboard queries use SQLAlchemy's defer() to exclude heavy columns (content, embedding, file_data) from listing queries, reducing payload sizes by ~95%.
  • Client-Side Caching: Dashboard stats are cached in sessionStorage to prevent redundant API calls on navigation. The cache is invalidated on upload or deletion events.
  • Connection Pooling: Uses SQLAlchemy's NullPool strategy to prevent connection exhaustion on Vercel's serverless infrastructure.

5. Upload Governance & Per-File Tagging

  • 10-Document Limit: The backend counts existing materials per user before processing uploads. If the new batch would exceed 10, the request is rejected with a descriptive error message.
  • Per-File Tags: The upload UI allows individual tag assignment per file. Tags are transmitted as a 2D JSON array ([["tag1"], ["tag2", "tag3"]]) mapped by index to each file.
  • Global Tag Injector: When uploading 2+ files, an "Apply Tag to All Files" input appears, instantly broadcasting a tag into every file's tag list.

6. Security & Authorization

  • JWT Authentication: All protected endpoints use OAuth2 bearer tokens signed with HS256. Tokens expire after 7 days.
  • Password Security: Passwords are hashed using bcrypt with auto-generated salts.
  • Ownership Authorization: Material deletion verifies that the requesting user's ID matches the material's user_id. Unauthorized deletions return HTTP 403.
  • Cascade Deletion: Deleting a material automatically removes all associated MaterialView records to prevent foreign key constraint violations.
07 — Deployment

Deployment Architecture

  • Platform: Vercel (Serverless)
  • Frontend: Next.js 16 app built and served via Vercel's Edge Network
  • Backend: FastAPI Python application deployed as a Vercel Serverless Function via @vercel/python runtime
  • Routing: Vercel rewrites proxy all /api/* requests to /api/main.py
  • Database: Supabase managed PostgreSQL instance with pgvector extension
  • Environment Variables: DATABASE_URL, JWT_SECRET, GEMINI_API_KEY
  • Package Manager: pnpm with --no-frozen-lockfile install strategy
  • Limitation: Vercel serverless functions enforce a 5MB maximum file upload size per request. Files exceeding this limit will be rejected at the infrastructure level before reaching the application code.
